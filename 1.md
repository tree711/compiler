# 项目定位

**TinyC 编译器**

支持：

```c
int a,b,c;

a = 3;
b = 4;

c = a + b * 2;

if(c > 5)
{
    c = c + 1;
}

while(c < 20)
{
    c = c + 1;
}
```

支持：

✅ 变量声明

✅ 整数

✅ 算术表达式

✅ 关系表达式

✅ if

✅ if-else

✅ while

✅ 块语句 {}

✅ 语义检查

✅ 三地址码

✅ 简单优化

✅ 伪汇编生成

# 四个人怎么分


## A（组长）

负责：

# Parser（语法分析）

### 任务1

设计文法

例如：

```bnf
Program -> DeclList StmtList

Decl -> int IdList ;

Stmt -> AssignStmt
      | IfStmt
      | WhileStmt

AssignStmt -> id = Expr ;

Expr -> Expr + Term
      | Expr - Term
      | Term

Term -> Term * Factor
      | Term / Factor
      | Factor
```

---

### 任务2

构建AST

例如：

```cpp
class ASTNode
{
};
```

派生：

```cpp
ExprNode
StmtNode
IfNode
WhileNode
AssignNode
```

---

### 任务3

递归下降分析器

例如：

```cpp
parseProgram()
parseStatement()
parseExpression()
```

---

### 任务4

语法错误恢复

例如：

```text
Missing ;
Unexpected token
```

---

### 输出

AST

例如：

```text
Assign
├── a
└── +
    ├── b
    └── 3
```

---

工作量：

约30%

属于核心模块。


```text

```

---

### 公共子表达式删除

---

# 并行关系

真正依赖关系：

```text
B(Lexer)
   ↓
A(Parser)
   ↓
C(Semantic)
   ↓
D(IR)
   ↓
D(Optimize)
   ↓
D(CodeGen)
```

但是实际开发可以这样拆：

```text
A
设计AST接口

B
开发Lexer

C
开发SymbolTable

D
开发IR结构
```

四个人同时开工。

## 三地址码格式

例如：

```cpp
(op,arg1,arg2,result)
```

```text
(*,c,d,t1)
(+,b,t1,t2)
(=,t2,,a)
```

---

这三个接口定下来后，

A、B、C、D四个人就能真正并行开发。
