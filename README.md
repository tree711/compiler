# TinyC 编译器

一个基于 C++ 实现的完整 TinyC 编译器，支持词法分析、语法分析、语义分析、中间代码生成、代码优化和汇编代码生成。

## 项目结构

```
compiler/
├── include/           # 头文件目录
│   ├── AST.h          # 抽象语法树定义
│   ├── CodeGenerator.h # 代码生成器接口
│   ├── IR.h           # 中间表示（Intermediate Representation）
│   ├── IRGenerator.h  # IR生成器接口
│   ├── Optimizer.h    # 优化器接口
│   ├── Symbol.h       # 符号表定义
│   └── Token.h        # 词法单元定义
├── src/               # 源文件目录
│   ├── CodeGenerator.cpp # 汇编代码生成实现
│   ├── IRGenerator.cpp   # IR生成实现
│   ├── Lexer.cpp     # 词法分析器
│   ├── Optimizer.cpp # 代码优化器
│   ├── Parser.cpp    # 语法分析器
│   ├── Semantic.cpp  # 语义分析器
│   └── main.cpp      # 编译器主程序
├── .gitignore        # Git忽略配置
├── Makefile          # 编译脚本
└── README.md         # 项目说明文档
```

## 编译流程

编译器遵循经典的多阶段编译架构：

```
源代码 (.c)
    ↓
┌──────────────┐
│  词法分析     │  Lexer: 将源代码转换为 Token 流
└──────────────┘
    ↓
┌──────────────┐
│  语法分析     │  Parser: 构建抽象语法树 (AST)
└──────────────┘
    ↓
┌──────────────┐
│  语义分析     │  Semantic: 符号表构建与类型检查
└──────────────┘
    ↓
┌──────────────┐
│  IR生成      │  IRGenerator: 生成三地址码中间表示
└──────────────┘
    ↓
┌──────────────┐
│  代码优化     │  Optimizer: 基本块优化（可选）
└──────────────┘
    ↓
┌──────────────┐
│  代码生成     │  CodeGenerator: 生成 x86-64 汇编代码
└──────────────┘
    ↓
汇编代码 (.s)
```

## 功能特性

### 支持的语法结构

| 特性 | 示例 |
|------|------|
| 变量声明 | `int a;`, `float b = 3.14;` |
| 赋值语句 | `a = 10;`, `b = a + 5;` |
| 算术表达式 | `a + b * c`, `(a - b) / 2` |
| 比较表达式 | `a == b`, `x > 0`, `y <= 100` |
| if-else 语句 | `if (x > 0) { ... } else { ... }` |
| while 循环 | `while (i < 10) { ... }` |

### 支持的数据类型

- `int` - 整型
- `float` - 浮点型

### 编译器选项

| 选项 | 说明 |
|------|------|
| `-o <文件>` | 指定输出文件名（默认 `output.s`） |
| `-v` | 详细模式，显示各阶段输出 |
| `-O` | 启用代码优化 |

## 编译与运行

### 1. 编译编译器

```bash
make
```

这将生成名为 `compiler` 的可执行文件。

### 2. 使用编译器

```bash
# 基本用法
./compiler <源文件>

# 指定输出文件
./compiler test.c -o program.s

# 详细模式（显示各阶段）
./compiler test.c -v

# 启用优化
./compiler test.c -O

# 组合选项
./compiler test.c -o program.s -v -O
```

### 3. 运行生成的汇编代码

生成的汇编代码可以使用 `gcc` 编译并运行：

```bash
# 编译汇编代码
gcc program.s -o program

# 运行程序
./program
```

### 输入文件示例 (`test.c`)

```c
int main() {
    int a = 10;
    int b = 20;
    int sum = a + b;
    return sum;
```

## 核心组件说明

### 1. Lexer（词法分析器）

负责将源代码文本分解为 Token 序列，支持：
- 关键字识别：`int`, `float`, `if`, `else`, `while`, `return`
- 标识符和数字常量
- 运算符和分隔符

### 2. Parser（语法分析器）

采用递归下降分析法，构建抽象语法树（AST），支持：
- 程序级结构分析
- 声明语句解析
- 表达式解析（支持运算符优先级）
- 控制流语句解析

### 3. Semantic Analyzer（语义分析器）

进行静态语义检查：
- 符号表管理
- 变量类型检查
- 作用域分析

### 4. IR Generator（中间代码生成器）

将 AST 转换为三地址码中间表示（IR）：
- 基于基本块的 IR 结构
- 支持常见算术和逻辑操作
- 控制流指令生成

### 5. Optimizer（优化器）

对中间代码进行优化：
- 常量传播
- 死代码消除
- 基本块优化

### 6. Code Generator（代码生成器）

将优化后的 IR 转换为 x86-64 汇编代码：
- 寄存器分配
- 栈帧管理
- 系统调用封装

## 技术栈

- **语言**: C++17
- **构建工具**: Make
- **目标平台**: x86-64 Linux/macOS
- **汇编格式**: AT&T 语法

- 头文件使用 `#pragma once` 保护
- 类和函数使用 Doxygen 风格注释
- 命名规范：类名使用 PascalCase，函数和变量使用 camelCase
- 使用智能指针管理动态内存

```bash

```

这将显示：
- Token 列表
- AST 结构
- 符号表信息
- 中间代码
- 生成的汇编代码
