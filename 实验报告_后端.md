## 一、后端总体架构

后端是整个编译器的后半部分，负责将经过词法分析、语法分析、语义分析后生成的 **AST（抽象语法树）** 转换为目标代码。本编译器的后端包含三个核心模块：

```
AST
 │
 ▼
┌─────────────┐
│ IRGenerator │  ← 中间代码生成（三地址码）
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Optimizer  │  ← 代码优化
└──────┬──────┘
       │
       ▼
┌───────────────┐
│ CodeGenerator │  ← 目标代码生成（x86-64 风格汇编）
└───────────────┘
       │
       ▼
    汇编文件 (.s)
```

### 模块依赖关系

| 模块 | 输入 | 输出 | 依赖 |
|------|------|------|------|
| IRGenerator | AST | IRProgram（三地址码） | AST.h, IR.h |
| Optimizer | IRProgram | IRProgram（优化后） | IR.h |
| CodeGenerator | IRProgram | 汇编文件 (.s) | IR.h |

## 二、IR（中间表示）设计

### 2.1 三地址码格式

每条指令采用 **四元式** 表示：

```
(opcode, result, operands)
```

- **opcode**：指令类型（枚举 `IROpcode`）
- **result**：运算结果（临时变量如 `%t1`，或变量名）
- **operands**：操作数列表

### 2.2 指令集

#### 算术运算

| opcode | 含义 | 示例 | 说明 |
|--------|------|------|------|
| `ADD` | 加法 | `%t3 = ADD %t1 %t2` | `%t3 = %t1 + %t2` |
| `SUB` | 减法 | `%t3 = SUB %t1 %t2` | `%t3 = %t1 - %t2` |
| `MUL` | 乘法 | `%t3 = MUL %t1 %t2` | `%t3 = %t1 * %t2` |
| `DIV` | 除法 | `%t3 = DIV %t1 %t2` | `%t3 = %t1 / %t2` |

#### 数据传送

| opcode | 含义 | 示例 | 说明 |
|--------|------|------|------|
| `CONST` | 常量定义 | `%t1 = CONST 3` | 将常量 3 存入 `%t1` |
| `LOAD` | 加载变量 | `%t1 = LOAD a` | 将变量 a 的值加载到 `%t1` |
| `STORE` | 存储变量 | `a = STORE %t1` | 将 `%t1` 的值写入变量 a |

#### 比较运算（本报告新增）

| opcode | 含义 | 示例 | 结果 |
|--------|------|------|------|
| `CMP_EQ` | 相等比较 | `%t3 = CMP_EQ %t1 %t2` | `%t1 == %t2` → 1，否则 0 |
| `CMP_NE` | 不等比较 | `%t3 = CMP_NE %t1 %t2` | `%t1 != %t2` → 1，否则 0 |
| `CMP_LT` | 小于比较 | `%t3 = CMP_LT %t1 %t2` | `%t1 < %t2` → 1，否则 0 |
| `CMP_GT` | 大于比较 | `%t3 = CMP_GT %t1 %t2` | `%t1 > %t2` → 1，否则 0 |
| `CMP_LE` | 小于等于 | `%t3 = CMP_LE %t1 %t2` | `%t1 <= %t2` → 1，否则 0 |
| `CMP_GE` | 大于等于 | `%t3 = CMP_GE %t1 %t2` | `%t1 >= %t2` → 1，否则 0 |

#### 控制流

| opcode | 含义 | 示例 | 说明 |
|--------|------|------|------|
| `BRANCH` | 无条件跳转 | `BRANCH label` | 跳转到 label |
| `BRANCH_EQ` | 条件跳转（相等） | `BRANCH_EQ %t1 1 label` | 若 `%t1 == 1` 则跳转 |
| `BRANCH_NE` | 条件跳转（不等） | `BRANCH_NE %t1 0 label` | 若 `%t1 != 0` 则跳转 |
| `BRANCH_LT` | 条件跳转（小于） | `BRANCH_LT %t1 0 label` | 若 `%t1 < 0` 则跳转 |
| `BRANCH_GT` | 条件跳转（大于） | `BRANCH_GT %t1 0 label` | 若 `%t1 > 0` 则跳转 |
| `BRANCH_LE` | 条件跳转（小于等于） | `BRANCH_LE %t1 0 label` | 若 `%t1 <= 0` 则跳转 |
| `BRANCH_GE` | 条件跳转（大于等于） | `BRANCH_GE %t1 0 label` | 若 `%t1 >= 0` 则跳转 |
| `RET` | 返回 | `RET %t1` | 返回 `%t1` 的值 |

### 2.3 基本块（BasicBlock）

基本块是连续的指令序列，具有单一入口和单一出口。控制流指令（BRANCH / BRANCH_EQ 等）只能出现在基本块末尾。

```
BasicBlock
├── name: "entry" / "then" / "while_cond" 等
├── instructions: IRInstruction 列表
├── successors: 后继基本块列表
└── predecessors: 前驱基本块列表
```

### 2.4 函数与程序

```
FunctionIR
├── name: 函数名
├── parameters: 参数列表
├── blocks: 基本块列表
└── entry: 入口基本块

IRProgram
├── functions: 函数列表
└── globalVariables: 全局变量列表
```

## 三、IRGenerator — 中间代码生成

### 3.1 设计思路

IRGenerator 负责遍历 AST（抽象语法树），为每个 AST 节点生成对应的三地址码指令。

核心流程：

```
IRGenerator::generate(ASTNode* root)
  └→ ProgramNode
       └→ FunctionDefinitionNode
            └→ 创建 FunctionIR
            └→ 创建入口 BasicBlock
            └→ BlockNode
                 └→ 对每个 statement:
                      ├→ AssignmentNode   → LOAD + 运算 + STORE
                      ├→ IfStatementNode  → CMP + BRANCH_EQ + BRANCH
                      ├→ WhileStatementNode → BRANCH + CMP + BRANCH_EQ
                      ├→ ReturnStatementNode → RET
                      └→ DeclarationNode  → (可选的) LOAD + STORE
```

### 3.2 关键算法

#### 临时变量分配

使用计数器生成唯一的临时变量名：

```cpp
std::string newTemp() {
    return "%t" + std::to_string(tempCounter++);
}
```

#### 表达式生成（递归）

```cpp
generateExpression(ASTNode* node) {
    switch (node->type) {
        case IDENTIFIER:  生成 LOAD 指令，返回临时变量
        case NUMBER:     生成 CONST 指令，返回临时变量
        case BINARY_OP:  递归生成左右子表达式 → 生成运算指令
        case UNARY_OP:   递归生成子表达式 → 生成取负/取反指令
    }
}
```

#### 控制流生成

**if-else 结构**：
```
         ┌─────────────────┐
         │   条件表达式     │
         │   cond = ...    │
         │   BRANCH_EQ cond│
         │     1, then     │
         │   BRANCH else   │
         └───────┬─────────┘
                 │
     ┌───────────┴───────────┐
     ▼                       ▼
┌──────────┐          ┌──────────┐
│  then    │          │  else    │
│  ...     │          │  ...     │
│ BRANCH   │          │ BRANCH   │
│ merge    │          │ merge    │
└────┬─────┘          └────┬─────┘
     └──────────┬──────────┘
                ▼
          ┌──────────┐
          │  merge   │
          │  ...     │
          └──────────┘
```

**while 循环结构**：
```
         ┌─────────────────┐
         │   BRANCH cond    │──────┐
         └─────────────────┘      │
                                  │
         ┌─────────────────┐      │
         │  while_cond     │◄─────┘
         │  cond = ...     │
         │  BRANCH_EQ cond │
         │    1, body      │
         │  BRANCH exit    │
         └───────┬─────────┘
                 │
     ┌───────────┴───────────┐
     ▼                       ▼
┌──────────┐          ┌──────────┐
│  body    │          │  exit    │
│  ...     │          │  ...     │
│ BRANCH   │          └──────────┘
│ while_   │
│ cond     │
└────┬─────┘
     │
     └─────────→ (返回 while_cond)
```

### 3.3 示例

输入 TinyC 代码：
```c
c = a + b * 2;
```

生成的 IR：
```
%t0 = LOAD a
%t1 = LOAD b
%t2 = CONST 2
%t3 = MUL %t1 %t2
%t4 = ADD %t0 %t3
c = STORE %t4
```

### 3.4 优化：关系表达式处理

在任务一中，我们发现原始 IRGenerator 不支持关系运算符（`>`, `<`, `>=`, `<=`, `==`, `!=`）。修复方法：

1. 在 `IROpcode` 枚举中新增 6 个比较指令（`CMP_EQ`, `CMP_NE`, `CMP_LT`, `CMP_GT`, `CMP_LE`, `CMP_GE`）
2. 在 `generateExpression` 的 `BINARY_OP` 分支中增加关系运算符的映射
3. 修复 `if`/`while` 的 `BRANCH_EQ` 指令格式（从 4 操作数改为 3 操作数 + 独立 BRANCH）

## 四、Optimizer — 代码优化

### 4.1 常量折叠（Constant Folding）

**原理**：在编译时计算常量表达式的值，避免运行时重复计算。

**算法**：
```
遍历所有基本块中的所有指令：
  如果指令是 ADD/SUB/MUL/DIV/CMP_*：
    检查所有操作数是否都是字面常量（不以 % 开头）
    如果是：
      计算运算结果
      将指令替换为 CONST 指令
```

**示例**：
```
优化前:  %t1 = ADD 3 4
优化后:  %t1 = CONST 7

优化前:  %t2 = CMP_GT 5 3
优化后:  %t2 = CONST 1    (5 > 3 为真)
```

### 4.2 复制传播（Copy Propagation）

**原理**：当一个临时变量只是另一个变量的别名时，直接用原始变量替换，减少不必要的中间变量。

**算法**：
```
维护一个映射表 copyMap (临时变量 → 原始变量)
遍历所有指令：
  如果是 LOAD:   记录 %t → 变量名 到 copyMap
  否则:
    将操作数中出现在 copyMap 中的替换为原始变量
    如果当前指令定义了新值，从 copyMap 中移除旧映射
```

**示例**：
```
优化前:  %t0 = LOAD c
         %t1 = CONST 20
         %t2 = CMP_LT %t0 %t1

优化后:  %t1 = CONST 20
         %t2 = CMP_LT c %t1    (LOAD 被消除)
```

### 4.3 死代码删除（Dead Code Elimination）

**原理**：删除计算结果从未被使用的指令，减少无用计算。

**算法**：
```
第一遍：标记所有"有副作用"的指令的操作数（STORE、BRANCH、RET）
迭代：标记被已标记指令使用的临时变量
删除：未被标记的指令（其计算结果未被使用）
```

**保留规则**：
- `STORE` 指令（写入变量，有副作用）→ 始终保留
- `BRANCH` / `BRANCH_*` / `RET`（控制流）→ 始终保留
- 其他指令仅当结果被使用时保留

**示例**：
```
优化前:  %t1 = CONST 1     ← 结果未被使用
         %t2 = CONST 2     ← 结果被 STORE 使用 → 保留
         a = STORE %t2

优化后:  %t2 = CONST 2
         a = STORE %t2      (%t1 = CONST 1 被删除)
```

### 4.4 优化流水线

优化器按以下顺序执行三个优化遍：

```
optimize(IRProgram& program)
  └→ 对每个函数:
       └→ constantFolding(func)   // 1. 常量折叠
       └→ copyPropagation(func)   // 2. 复制传播
       └→ deadCodeElimination(func) // 3. 死代码删除
```

这种顺序确保了前一优化的结果可以为后一优化创造更多机会（例如常量折叠后产生未使用的临时变量，正好被死代码删除清除）。

## 五、CodeGenerator — 目标代码生成

### 5.1 设计思路

CodeGenerator 将优化后的三地址码转换为 x86-64 风格的汇编代码。采用**栈式分配**策略管理临时变量。

### 5.2 指令映射

| IR 指令 | 汇编指令 | 说明 |
|---------|---------|------|
| `CONST %t, v` | `movl $v, %eax` / `pushl %eax` | 将常量压栈 |
| `LOAD %t, var` | `movl var, %eax` / `pushl %eax` | 加载变量到栈 |
| `STORE var, %t` | `movl offset(%rbp), %eax` / `movl %eax, var` | 弹栈到变量 |
| `ADD %t, a, b` | `movl a, %eax` / `addl b, %eax` / `pushl %eax` | 加法 |
| `SUB %t, a, b` | `movl a, %eax` / `subl b, %eax` / `pushl %eax` | 减法 |
| `MUL %t, a, b` | `movl a, %eax` / `imull b` / `pushl %eax` | 乘法 |
| `DIV %t, a, b` | `movl a, %eax` / `idivl b` / `pushl %eax` | 除法 |
| `CMP_XX %t, a, b` | `cmpl b, %eax` / `setX %al` / `movzbl %al, %eax` | 比较，结果 0/1 |
| `BRANCH label` | `jmp label` | 无条件跳转 |
| `BRANCH_EQ cond, v, l` | `cmpl $v, %eax` / `je l` | 相等跳转 |
| `BRANCH_NE cond, v, l` | `cmpl $v, %eax` / `jne l` | 不等跳转 |
| `BRANCH_LT cond, v, l` | `cmpl $v, %eax` / `jl l` | 小于跳转 |
| `BRANCH_GT cond, v, l` | `cmpl $v, %eax` / `jg l` | 大于跳转 |
| `BRANCH_LE cond, v, l` | `cmpl $v, %eax` / `jle l` | 小于等于跳转 |
| `BRANCH_GE cond, v, l` | `cmpl $v, %eax` / `jge l` | 大于等于跳转 |
| `RET %t` | `movl %eax, offset(%rbp)` / `popq %rbp` / `ret` | 函数返回 |

### 5.3 函数框架

每个函数生成以下汇编框架：

```asm
.globl function_name
function_name:
    pushq %rbp          # 保存栈基址
    movq %rsp, %rbp     # 设置栈帧
    ...                 # 函数体
    popq %rbp           # 恢复栈基址
    ret                 # 返回
```

### 5.4 临时变量管理

临时变量通过栈来管理：
- 分配：`pushl %eax` — 将结果压栈
- 访问：`movl offset(%rbp), %eax` — 通过偏移量读取
- 偏移量保存在 `variableOffsets` 映射表中

## 七、遇到的问题与解决方案

### 问题 1：关系运算符缺失

**问题描述**：原始 IRGenerator 不支持 `>`, `<`, `>=`, `<=`, `==`, `!=` 等关系运算符，遇到关系表达式时返回空字符串，导致 `if`/`while` 条件无法正确生成 IR。

**解决方案**：
1. 在 `IROpcode` 枚举中新增 `CMP_EQ`, `CMP_NE`, `CMP_LT`, `CMP_GT`, `CMP_LE`, `CMP_GE` 六个比较指令
2. 在 `generateExpression` 中增加运算符映射
3. 对应在 CodeGenerator 中增加 `sete`/`setne`/`setl`/`setg`/`setle`/`setge` 条件设置指令的生成

### 问题 2：BRANCH_EQ 指令操作数不匹配

**问题描述**：IRGenerator 中 `BRANCH_EQ` 传入了 4 个操作数（`cond`, `"1"`, `thenBlock`, `elseBlock`），但 CodeGenerator 只处理了 3 个操作数（缺少 elseBlock 的处理）。

**解决方案**：将 `if`/`while` 的 IR 生成改为 3 操作数 `BRANCH_EQ` + 独立的无条件 `BRANCH` 指令，语义更清晰且与 CodeGenerator 一致。

```
修改前:  BRANCH_EQ cond 1 thenBlock elseBlock
修改后:  BRANCH_EQ cond 1 thenBlock
         BRANCH elseBlock
```

### 问题 3：Optimizer 函数签名不匹配

**问题描述**：优化器的私有方法声明为 `IRProgram&` 参数，但在 `optimizeFunction` 中传入的是 `FunctionIR&`，导致编译错误。

**解决方案**：将所有私有方法的参数类型改为 `FunctionIR&`，并去掉实现中遍历 `program.functions` 的外层循环。

### 问题 4：常量折叠不支持比较运算

**问题描述**：原始的常量折叠只处理了 `ADD`/`SUB`/`MUL`/`DIV`，新增的 `CMP_*` 指令无法折叠。

**解决方案**：在常量折叠中增加对 `CMP_*` 指令的处理，`3 > 5` → `0`，`2 == 2` → `1` 等。

### 问题 5：死代码删除范围过小

**问题描述**：原始的 DCE 只在单个基本块内分析，且未保留 `BRANCH` 系列指令，导致跨基本块的优化不完整。

**解决方案**：改为跨基本块分析，同时显式保留所有有副作用的指令（`STORE`、`BRANCH*`、`RET`）。

## 八、总结

本实验完成了 TinyC 编译器的后端部分，包括：

1. **IRGenerator**：将 AST 转换为三地址码中间表示，支持算术运算、关系运算、if-else 条件分支、while 循环等控制结构
2. **Optimizer**：实现了常量折叠、复制传播、死代码删除三种优化算法，能够有效化简中间代码
3. **CodeGenerator**：将优化后的中间代码转换为 x86-64 风格的汇编代码

通过 7 个测试用例验证了各个模块的正确性，完整流水线测试展示了从 AST 到汇编的完整编译过程。
